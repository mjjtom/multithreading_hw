## Домашнее задание для лекции по теме "lock-free, средства межпроцессного взаимодействия"

Вам необходимо реализовать lock-free MPSC очередь сообщений, которая будет позволять обмениваться произвольными сообщениями между разными процессами.

Очередь должна быть реализована на основе кольцевого буффера и располагаться в разделяемой памяти.

Для того, чтобы дать возможность передавать произвольные сообщения необходимо зафиксировать свой протокол. Предлагается следующая идея:
* Все сообщения будут начинаться с одинакового заголовка, который будет содержать тип сообщения и его длину
* В разделяемой памяти помимо самой очереди также будет хранится некоторая мета-информация о протоколе, в частности его версия. Это позволит не допустить того, что разные процессы будут использовать разные версии протокола

Для использования очереди предлагается создать две основные сущности:
* ProducerNode -- позволяет писать произвольные сообщения в очередь. На плечи ProducerNode также возлагается обязанность по созданию и инициализации очереди
* ConsumerNode -- позволяет читать сообщения из очереди. При этом для ConsumerNode должен быть механизм, который будет позволять читать только сообщения заданного типа, а остальные отбрасывать

Очередь описывается некоторым путем, аналогично перовму аргменту shm_open и ее размером.

Необходимо будет продемонстрировать работоспособность очереди путем создания двух разных процессов(одна программа с fork() не подойдет), которые будут обмениваться сообщениями.
